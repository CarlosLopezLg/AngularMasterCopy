Routing Module


A continuación se presentarán las principales directivas de Angular.


¿Qué es el archivo de rutas?


Si accedes al archivo 'app.component.html', notarás que existe un componente llamado <router-outlet></router-outlet>. Este componente tiene como propósito cargar de manera dinámica distintos componentes, los cuales se generan dependiento de la ruta del navegador. Es por esto que Angular se considera un framework SPA (Single Page Application), ya que no necesita recargar la página para cargar otra página (en este caso, un componente).

Si accedes al archivo llamado 'app-routing.module.ts', notarás que existe un arreglo llamado 'routes', y es aquí en donde relacionarás una ruta con un componente.

{{ codeRoutes }}

Esta es una estructura simple de un arreglo de rutas, en donde predominan los siguientes puntos:


path: Aquí se incluye la ruta que cargará el componente.

component: Aquí se incluye el componente a cargar. Recuerda que el componente debe estar importado.

redirectTo: Como su nombre lo dice, te permite redireccionarte a otra ruta.


Observa que en el primer objeto, la variable path está vacía, esto indica la ruta original del sitio web (o localhost). También observa que en el último objeto, la variable path tiene como valor "**", esto es un comodín que hace referencia a todas aquellas rutas ingresadas y que no fueron asignadas previamente, por lo que siempre se debe colocar hasta el final.

existen más opciones que se pueden incluir en este arreglo, como los guards y lazy load que se analizarán más adelante.

Crear archivo de rutas


El archivo de rutas principal es 'app-routing.module.ts', pero puedes crear otro un módulo junto con un archivo de rutas con el siguiente comando:

ng generate module module-name --routing
